# EduInsight项目开发规范和流程

## 1. 开发顺序

### 后端优先原则
- **先实现后端，再实现前端**
- 后端完成后，必须运行 `npm run openapi2ts` 生成TypeScript类型
  - 该命令读取: `http://localhost:5000/openapi/openapi.json`
  - 自动生成类型定义到: `src/api/typings.d.ts`
  - 类型生成完成后才能开始前端实现

## 3. 实现文档维护

### 文档位置
- 在项目根目录维护唯一的实现文档
- 文档名称格式: `[模块名称]实现文档.md`
  - 例如: `成绩管理模块实现文档.md`
  - 例如: `学情预警与预测模块实现文档.md`
  - 例如: `统计分析模块实现文档.md`

### 更新时机
- 每实现一个功能后更新
- 每次修复bug后更新
- 每次需求调整后更新

### 文档内容包括
- 功能实现的具体原理
- 技术架构和设计
- 困难遇到与解决方案
- 变更记录

## 4. 前端导航方式

### 当前阶段
- 通过URL直接输入访问页面
- 不使用按钮跳转方式
- 后续整合时会统一调整

## 5. 错误处理和文档更新

### 困难处理流程
- 遇到困难时，在文档中添加"困难遇到与解决"专项
- 同一个困难可能需要多次尝试解决
- 只有在用户确认"问题解决了"后，才填写最终解决方案
- 每次调整都要同步更新实现文档

### 文档更新原则
- 保持文档与代码实现同步
- 记录所有重要的技术决策
- 记录问题和解决方案，便于后续参考
